Basic Gameplay and Combat

In the 3D videogame project Celery, I was responsible for coding most player related systems

Players can play both with keyboard and mouse or gamepad. Aim sensibility can also be adjusted in gameplay settings.

Actions include isometric movement, dash that can go over the void, melee hit, cooking system (main mechanic) and special skills for each food type.

Health System Movement and dash Combat and Skills Camera and cinematics

World Logic

As a dungeon crawler, the rooms logic is an important asset. A sophisticated manager was created to control enemy waves, room doors logic, room visibility, lighting and checkpoints.

The room manager, which contains one or several wave managers, is programmed to be an easy tool for both the level designer and the environment designer

Boss Fight

From the overall boss sequence to the individuals skills. The first boss combine both world logic and new specifically implemented attacks.

User Interface and Settings

The game features a mixed 2D-3D main menu with camera movements and transitions between menu options. Other interfaces include pause menu, level end menu, HUD and dialogue.

A data-persistant system is created to generate a recipe book to display player's statistics and progress. Also, a saving system generates a json file and loads it to restore the user's progress or configuration when needed.

Localization in 3 languages: Catalan, Spanish and English; interactive tutorial, audio management, score system and feedback screens such as the loading screen.
